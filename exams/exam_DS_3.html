Data Structures Exam 3

  1. Implement an array data structure with the following functions in O(1):

        1. void setval(int indx, int val); 
    2. int getval(int indx);
    3. void setall(int val);  (no memset);

  2. Given an array of unsigned ints and some number “sum_of_two”, write a function finding two array values that gives the sum of the given number (sum_of_two). Also write the time complexity of your solution.

  3. Implement a “find” function that receives an array of chars and size of the array. The function checks whether the char is in the array. You are not allowed to use if nor the ternary operator.

  4. Given an array of chars, write a function that performs an efficient n bytes circular shift in-place (don't do a 1 circular shift n times).

  5. Given a 2D bitmap of 1s and 0s, 0 represents sea and 1 represents shore. Implement a function finding how many islands the map contains.

  6. We use a stack for saving our data – “data_stack”. Design a stack “min_stack” which allows us to get the minimum value of data_stack in O(1).

  7. Write a function that receives a string, checks whether its parentheses ( (), [], {}, <>) are arranged correctly, and returns 0 or 1. Example:

    • "(x + 3 * [4 + 6]) <>" would return 1
    • "(8 ]* (6 + 2) + 1)" would return 0
    • "[(8 ]* (6 + 2) + 1)" would return 0

  8. Recursion:

    a. Give an example of a recursive function. What happens if you don't write a stop condition?

    b. Write a recursive function which does arithmetic operations such as multiplication and division of two integers without using * or /.

    c. Write a recursive function which receives an integer and returns its argument incremented by 1. You may not use addition.

  9. Sort:

    a. Which sorting algorithms do you know? Specify the average time complexity and memory complexity for each.

    b. What is a stable sort? Which sorts are stable?

    c. Write a function that receives a linked list and a pivot value x, and arranges the linked list according to the pivot (pivot does not have to be equal to one of the list’s elements). Each value bigger or equal to x will appear after x, and every value smaller will appear before x.

    d. You are given 100K text files. Each has a sorted array of 10 numbers. Your goal is to combine all 100K arrays into one sorted array.
    • What sorting algorithm would you choose? Why?
    • What is the bottleneck in the program?

  10. Given a dictionary, find a quick way to find all anagrams of a given word that are valid dictionary words. You can prepare in advance any required code or data structure.

    For example, live, vile, veil and evil are anagrams.

  11. Design the data structures and interfaces for a hash table:

    a. Write the interface for a hash table.

    b. Describe the time complexity of the relevant functions.

    c. Give an example of when you would use a hash table.

  12. Design the data structures and interfaces for a heap:

    a. Describe the purpose of the heap data structure. Give an example of when you would use it.

    b. Write the interface for a heap.

    c. Describe the time complexity of the relevant functions.

  13. Binary tree, binary search, IP:

    a. Implement an iterative and recursive insertion to a binary tree.

    b. What is the complexity of insert to a binary search tree?

    c. In the IPv4 protocol, the IP address belongs to some geographical region, and its format is: xxx.xxx.xxx.xxx. For example:

    • All IP addresses between 10.7.8.10 and 10.7.20.250 belong to Eilat, Israel. Thus 10.7.13.62 is an address in Eilat, Israel.
    • IP addresses between 10.8.9.9 and 10.8.20.40 belong to Rishon-Le-Zion, Israel.

    d. Offer a way to store the data, so that given an IP address, you can efficiently discover to which city it belongs.

  14. You are given a map of roads and cities. A road is always straight, has a length, and connects two cities, bi-directionally.

    • Offer a data structure to store the information about the map.
    • The data structure should allow an efficient search for a direct connection between two cities, to be used by algorithms for solving the shortest path problem, for example.
